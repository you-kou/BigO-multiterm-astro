格雷码（Gray Code），也被称为**循环二进制码**或**反射二进制码**。它最大的特点是：**任意两个相邻的数值，它们的二进制表示中只有一位数（比特）是不同的。**

这个特性在计算机科学、数字电路和通信领域非常重要，能解决很多普通二进制编码会带来的“卡顿”和错误问题。

## 为什么要用格雷码？

假设我们在一个转盘上安装了光电传感器，用来读取转盘的角度。我们让它从数字 `3` 转到 `4`。

- **普通二进制（011 ➔ 100）：三位全变。** 由于物理电路有快慢，在三个位置同时翻转的瞬间，计算机极易读出 `111`（7）或 `000`（0）等错误的中间值（称为**竞争冒险**）。
- **格雷码（010 ➔ 110）：仅变一位。** 只有最左边的一位从 0 变 1，其他两位完全没动。即使电路有轻微延时，也只会读出 3 或 4，**绝不会产生莫名其妙的错误干扰**。

## 格雷码与普通二进制对照表

我们可以通过一个表格直观地看出两者的区别：

| **十进制数** | **普通二进制码** | **格雷码** | **变化位数（对比上一行）** |
| ------------ | ---------------- | ---------- | -------------------------- |
| **0**        | `000`            | `000`      | -                          |
| **1**        | `001`            | `001`      | 格雷码只变了最后一位 ✓     |
| **2**        | `010`            | `011`      | 格雷码只变了倒数第二位 ✓   |
| **3**        | `011`            | `010`      | 格雷码只变了最后一位 ✓     |
| **4**        | `100`            | `110`      | 格雷码只变了第一位 ✓       |
| **5**        | `101`            | `111`      | 格雷码只变了最后一位 ✓     |
| **6**        | `110`            | `101`      | 格雷码只变了倒数第二位 ✓   |
| **7**        | `111`            | `100`      | 格雷码只变了最后一位 ✓     |

**循环性：** 注意看最后一个数字 `7` (`100`) 和第一个数字 `0` (`000`)，它们之间也仅仅相差一位。这就是为什么它被称为“循环码”，非常适合用来表示圆形的转盘（编码器）。

## 反射定律

格雷码的英文叫 *Reflected Binary Code*（反射二进制码），因为它的构建过程就像照镜子一样：

### 第一步：从“1位”开始

我们手头只有最基础的两个数字：

Plaintext

```
0
1
```

### 第二步：生成“2位”（照镜子）

现在，我们在 `0` 和 `1` 的下面**放一面镜子**（这条横线就是镜子）。

镜子里反射出来的倒影，一定是离镜子最近的先出现，对吧？所以 `1` 在上，`0` 在下：

```
  0
  1
—————— (我是镜子)
  1  (这是1的倒影)
  0  (这是0的倒影)
```

接下来，我们要让它们变成 2 位数。规则非常简单：

- **镜子上面的，前面全部补 0**
- **镜子下面的（倒影），前面全部补 1**

```
0 0
0 1
——————
1 1
1 0
```

看！这就是 **2位格雷码**：`00`, `01`, `11`, `10`。

### 第三步：生成“3位”（再照一次镜子）

我们把刚刚得到的 4 个数字当作一个整体，在下面**再放一面大镜子**：

```
  00
  01
  11
  10
—————————— (大镜子)
  10  (10的倒影)
  11  (11的倒影)
  01  (01的倒影)
  00  (00的倒影)
```

同样的方法，让它们变成 3 位数：

- **镜子上面的（本体），前面全补 0**
- **镜子下面的（倒影），前面全补 1**

```
0 00
0 01
0 11
0 10
——————
1 10
1 11
1 01
1 00
```

大功告成！去掉空格，这就变成了你在对照表里看到的 **3位格雷码**：`000`, `001`, `011`, `010`, `110`, `111`, `101`, `100`。

![n 位格雷码](./n-Bit-Gray-Code.png 'n 位格雷码')

### 为什么这种方法能保证“只变一位”？

因为是“照镜子”生成的，所以**紧挨着镜子上下两侧的两个数字，除了最前面新补的 `0` 和 `1` 不同之外，后面的部分是一模一样的倒影。**

比如第三步里，镜子上面的 `010` 和下面的 `110`，它们的后两位都是 `10`，仅仅是第一位不同。这就完美实现了“相邻两个数只相差一位”的核心特点！

## 普通二进制和格雷码之间互相转换

在位运算中，普通二进制（Binary）和格雷码（Gray Code）之间的互相转换非常高效，核心全靠一个大名鼎鼎的位运算符：**异或（XOR，符号为 `^`）**。

为了更好地理解，我们先复习一下异或的口诀：**相同为 0，不同为 1**（即 `1 ^ 1 = 0`, `0 ^ 0 = 0`, `1 ^ 0 = 1`）。

### 二进制 ➔ 格雷码

把一个二进制数变成格雷码，公式是：

$$G = B \oplus (B \gg 1)$$

- **大白话：** 把二进制数 $B$ **向右移动一位**（右移后高位补 0），然后和**原来的二进制数 $B$** 进行按位异或运算。
- **特性：** 格雷码的最高位（最左边那一位）和二进制的最高位是完全一样的。

#### 手算演示

以二进制数 `1010`（十进制的 10）为例：

```
  1 0 1 0  (原二进制数 B)
^   1 0 1  (右移一位后的 B >> 1，高位补0其实是 0101)
——————————
  1 1 1 1  (结果就是格雷码 G)
```

### 格雷码 ➔ 二进制

把格雷码变回二进制稍微有点像“连环套”。它的二进制最高位等于格雷码最高位，之后的每一位二进制数，都是**前一个算出来的二进制位**与**当前格雷码位**进行异或。

在写代码时，有一个非常巧妙的数学等价公式：

$$B = G \oplus (G \gg 1) \oplus (G \gg 2) \oplus (G \gg 4) \oplus \dots$$

- **大白话：** 把格雷码不断向右移位（1位、2位、4位、8位...），然后把所有的结果全都异或起来，直到右移后的数字变成 0 为止。

#### 手算演示

以格雷码 `1111` 为例：

```
  1 1 1 1  (原格雷码 G)
^   1 1 1  (G >> 1)
^     1 1  (G >> 2)
^       1  (G >> 3)
——————————
  1 0 1 0  (结果成功还原回二进制 B)
```

## 常见应用场景

- **绝对值编码器：** 工业机器人、数控机床中用来测量电机旋转角度的硬件。
- **异步FIFO（先进先出队列）：** 在数字电路（如FPGA/ASIC）中跨时钟域传递指针时，用格雷码可以防止数据采样出错。
- **卡诺图化简：** 逻辑电路设计中，卡诺图的坐标排列必须使用格雷码。
